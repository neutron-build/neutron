# Security Policy

## Reporting A Vulnerability

Do not open public GitHub issues for suspected security vulnerabilities.

Report privately with:

- affected package(s)
- affected version(s)
- reproduction steps or proof of concept
- impact assessment

Project maintainers will acknowledge receipt within 72 hours and provide a remediation timeline after triage.

## Disclosure Process

1. Triage and severity assessment.
2. Fix prepared in a private branch.
3. Coordinated release of patched versions.
4. Public advisory/changelog note with mitigation guidance.

## Supported Versions

- Current major line: fully supported.
- Previous major line: security fixes for 12 months after the next major release.
- Older lines: unsupported.
